# 비즈니스 요구사항 문서 (BRD)

## 프로덕트 이름

작업명: `FutureKey`  
대체 컨셉명: `미래 예약형 메시지/메모리 전달 플랫폼`

## 문서 버전

Version 2.0 (통합본)  
작성일: 2026년 4월 20일  
언어: 한국어  
대상: 창업팀 / 아이데이션 논의용

---

## 1. 문서 목적

이 문서는 **미래 예약형 메시지/영상 전달 플랫폼 (FutureKey)** 의 비즈니스 요구사항을 정의하기 위한 통합 초안이다.

본 서비스는 법적 효력이 있는 유언장 서비스가 아니라, 사용자가 미래의 특정 시점에 맞춰 **텍스트 메시지, 영상 메시지, 음성 메모, 사진** 등을 특정인에게 전달할 수 있도록 돕는 감성 기반 플랫폼을 목표로 한다. 향후에는 소형 선물 또는 기프트카드 전달 기능으로 확장한다.

이 BRD의 목적은 아래와 같다.

- 서비스의 비즈니스 방향성 정리
- 해결하려는 문제와 사용자 가치 정의
- 핵심 프로덕트 구조 (키 기반 접근 제어) 정의
- 무료/유료 모델 구분
- MVP 범위 확정 전 논의 기준 마련
- 이후 PRD 작성의 기반 제공

---

## 2. 서비스 개요

### 2.1 서비스 한 줄 정의

사용자가 미래의 특정 날짜에 맞춰 메시지나 영상을 남기고, 지정한 상대가 그 시점에 열람할 수 있도록 하는 **예약형 감성 플랫폼**.

### 2.2 핵심 컨셉

- 지금의 감정, 진심, 기록을 미래의 특정 시점에 전달
- 크리에이터는 복수의 **키(Key)** 를 생성하고, 콘텐츠를 하나 이상의 키에 할당
- 수신자는 "무언가가 나에게 남겨져 있다"는 사실은 알 수 있지만, 실제 내용은 지정된 시점 전까지 열람 불가
- 링크 또는 권한 기반으로 일부 공개 정보를 보여줄 수 있어 기대감과 바이럴 요소를 형성
- 향후에는 작은 선물, 기프트카드 등의 실행 기능을 추가해 감정 전달 + 실용적 전달까지 확장

### 2.3 법적 포지셔닝

본 서비스는 **유언장 작성/집행 서비스가 아니며**, 법적 상속, 증여, 신탁, 유산 분배를 직접 다루지 않는다.  
초기 서비스는 아래 범위에 집중한다.

- 개인 메시지 (텍스트, 영상, 음성, 사진)
- 일정 시점 기반 열람 제어
- 선택적 수신자 알림
- 향후 소규모 디지털 또는 경량 선물 전달

서비스의 본질은 **법률 서비스가 아니라 감정 전달 및 미래 예약 커뮤니케이션 서비스**이다.

## 3. 문제 정의

### 3.1 해결하려는 문제

사람들은 미래의 누군가에게 지금의 마음을 남기고 싶어하지만, 이를 안정적으로 기록하고 특정 시점에 전달할 수 있는 구조는 제한적이다.

대표적인 문제는 다음과 같다.

1. 미래의 특정 날짜에 맞춘 감정 전달 수단이 부족하다.
2. 영상/텍스트를 안전하게 보관하고 특정 시점까지 잠그는 경험이 흔하지 않다.
3. 단순 저장이 아니라 "전달될 예정임"을 보여주는 경험이 부족하다.
4. 기존 메시징 앱은 즉시 전달 중심이며, 시간 기반 제어가 없다.
5. 클라우드 저장소는 기능적이지만 감성적 UX가 없고 구조화되지 않았다.
6. 소셜 플랫폼은 공개 중심이라 친밀한 레거시 콘텐츠에 적합하지 않다.
7. 유언/레거시 서비스는 법적/사망 이후 맥락에 치우쳐 있어, 일상적이고 가벼운 미래 메시지 시장과는 결이 다르다.

### 3.2 시장 기회

본 서비스는 아래 카테고리의 중간 지점을 노릴 수 있다.

- 디지털 메모리 / 타임캡슐 서비스
- 개인 감정 기록 서비스
- 예약형 메시지 전달 서비스
- 경량형 레거시/메시지 플랫폼
- 향후 감성 커머스(기프트카드/소형 선물) 확장 플랫폼

특히 차별점은 **"남겨져 있다는 사실을 보여주되, 내용은 잠금"** 하는 구조와,  
**무료-유료 이원화 모델**, 그리고 **미래 전달의 기대감과 소셜성**에 있다.

## 4. 비즈니스 목표

### 4.1 단기 목표

- 감성적이고 간단한 미래 메시지 예약 플랫폼의 MVP 출시
- 사용자 반응 검증
- 핵심 UX (남김, 잠금, 열람, 기대감, 공유 가능성) 검증
- 무료 유입 구조와 유료 전환 포인트 정의

### 4.2 중기 목표

- 보안 저장 기능을 중심으로 프리미엄 플랜 정착
- 링크 기반 공개/부분 공개 기능을 통한 바이럴 유입 확보
- 인플루언서/크리에이터/셀럽 활용 가능성 탐색
- 가족, 연인, 친구, 부모-자녀 등 감정 기반 시나리오 강화

### 4.3 장기 목표

- 디지털 메시지에서 소형 선물/기프트카드 실행으로 확장
- 신뢰 기반 미래 전달 플랫폼으로 포지셔닝
- 감정 기록 + 전달 + 기념일/이벤트 실행을 포괄하는 플랫폼으로 성장

---

## 5. 프로덕트 비전 및 목표

### 5.1 비전

개인 메시지, 기억, 레거시 콘텐츠를 미래에 남기기 위한 가장 단순하고 신뢰할 수 있는 플랫폼이 된다.

### 5.2 프로덕트 목표

사용자가 감정적으로 의미 있는 콘텐츠를 만들 수 있도록 한다:

- 특정 수신자에게 키(Key)를 통해 할당됨
- 선택한 날짜까지 잠긴 상태를 유지
- 공개 또는 비공개로 공유 가능
- 비기술적인 사용자도 사용할 수 있을 만큼 단순한 경험

경험 전체는 극도로 단순하고, 따뜻하며, 신뢰할 수 있어야 한다.

## 6. 대상 사용자 및 페르소나

### 6.1 Primary User (메시지 생성자 / 크리에이터)

미래에 메시지 또는 영상을 남기고 싶은 사람

예시:
- 자녀에게 미래 메시지를 남기고 싶은 부모
- 특정 기념일에 맞춰 메시지를 예약하려는 연인 또는 배우자
- 생일이나 졸업 시점에 영상을 남기려는 친구
- 질병이나 불확실한 상황에 처한 사람이 미리 메시지를 준비하는 경우
- 셀럽/크리에이터가 팬 또는 특정 대상에게 메시지를 남기고 존재만 공개하는 경우

### 6.2 Secondary User (수신자 / Recipient)

미래의 특정 시점에 메시지를 받는 사람

예시:
- 자녀, 배우자/연인, 친구, 가족
- 특정 커뮤니티 멤버, 팬/구독자/지인

### 6.3 Viewer / Audience (선택적 공개 대상)

내용은 볼 수 없지만, 메시지가 존재함을 알 수 있는 사람

예시:
- 링크를 가진 사람
- 지정된 일부 사용자 그룹
- 프로필 페이지 방문자
- 여러 수신자 중 공동 열람 예정자

## 7. 핵심 가치 제안

### 7.1 사용자 가치

- 미래의 나/타인에게 진심을 남길 수 있다.
- 지금 이 순간의 감정을 안전하게 저장할 수 있다.
- 지정 시점 전까지 내용이 잠겨 있어 기대감을 만든다.
- 누군가가 나에게 메시지를 남겼다는 사실만으로도 감정적 연결이 형성된다.
- 단순 저장이 아닌 "전달 예정" 경험을 제공한다.

### 7.2 서비스 차별점

- 유언장이 아니라 일상적인 미래 메시지 중심
- 영상/텍스트/음성/사진 모두 가능
- **존재 공개 / 내용 비공개 UX** — 단순 타임캡슐이 아니라, 기대감과 긴장감을 만드는 UX
- 키(Key) 기반 세밀한 접근 제어 (라벨링, 복수 키, 키 취소)
- 무료 진입이 쉬움
- 프리미엄은 보안 저장 및 전달 신뢰성 강화
- 장기적으로는 작은 선물 실행 기능까지 확장 가능

## 8. 주요 비즈니스 시나리오

### 시나리오 1: 부모 → 자녀
부모가 자녀에게 10년 뒤 열어볼 영상 메시지를 남긴다.  
자녀는 "나에게 미래 메시지가 있다"는 사실을 알 수 있지만, 지정일 전까지 열람할 수 없다.

### 시나리오 2: 연인/배우자
기념일이나 결혼기념일에 맞춰 메시지/영상 예약 발송.

### 시나리오 3: 친구/가벼운 관계
생일, 졸업, 입학 등 특정 이벤트에 맞춘 메시지 예약.

### 시나리오 4: 공개 기대감 유도
사용자가 자신의 페이지에 "누구에게 어떤 시점의 메시지를 남겼다"는 사실만 공개.  
방문자들은 내용을 볼 수 없지만, 존재를 인지하며 호기심과 기대감을 느낀다.

### 시나리오 5: 복합 키 사용
할머니가 손녀를 위해 레시피 영상을 녹화하고, "가족" 키와 "손녀" 키 두 개에 모두 할당. 한쪽 키 소유자가 잠금 해제 후 콘텐츠를 열람.

### 시나리오 6: 향후 선물 연계
메시지와 함께 특정 시점에 기프트카드 또는 소액 선물 전달.

## 9. 프로덕트 원칙

- 일반 사용자도 사용할 수 있을 만큼 단순하게
- 설계 자체가 프라이버시 중심
- 소셜 참여보다 감정적 신뢰 우선
- 접근 제어는 법적 관계 확인이 아닌 크리에이터 의도에 따름
- 시간 기반 잠금 해제는 핵심 프로덕트 기능
- 법적 유언 기능은 MVP 범위 밖

## 10. 범위 (Scope)

### 10.1 MVP 포함 항목

- 사용자 계정 생성 및 로그인, 보안 계정 복구
- 복수 키(Key) 생성 및 관리 (라벨 지정 가능)
- 영상, 음성, 텍스트, 사진 업로드 (내부 저장 또는 외부 링크)
- 콘텐츠를 하나 이상의 키에 할당
- 콘텐츠 공개/비공개 설정
- 잠금 해제 날짜/시간 설정 (즉시 또는 미래 시점)
- 잠긴 콘텐츠의 목록 표시 (내용은 비공개)
- 키 소유자는 자신의 키에 연결된 콘텐츠만, 잠금 해제 후 열람 가능
- 크리에이터의 키/접근 코드 공유 기능
- 콘텐츠 잠금 해제 시 기본 알림 (이메일, 푸시, 인앱)
- 콘텐츠 라이브러리 및 타임라인/목록 인터페이스
- 기본 프로필/페이지 및 공개 범위 설정
- 무료/유료 기본 구분 구조

### 10.2 MVP 제외 또는 후순위 항목

- 법적 유언 작성 또는 법적 검증
- 공식 가족 관계 확인
- 변호사, 공증, 유산 정리 워크플로우
- 실물 선물 구매 및 배송
- 복잡한 마켓플레이스 또는 광고 연동
- AI 편집 또는 전사 (MVP 핵심 의존성으로 포함 안 함)
- 공개 소셜 피드 또는 커뮤니티 기반 콘텐츠 발견

## 11. 핵심 기능 요구사항

### 11.1 사용자 계정

- 회원가입, 로그인, 개인 프로필 관리
- 개인 메시지 볼트(Vault) 생성 및 관리
- 보안 계정 복구 수단 제공

### 11.2 키(Key) 관리

- 크리에이터는 복수의 키를 생성할 수 있어야 한다.
- 각 키에는 "딸", "배우자", "가족" 등의 라벨을 붙일 수 있어야 한다.
- 크리에이터는 하나 이상의 콘텐츠를 하나 이상의 키에 할당할 수 있어야 한다.
- 크리에이터는 키 또는 접근 코드를 수신자에게 공유할 수 있어야 한다.
- 시스템은 보호된 콘텐츠 접근 전에 유효한 키 보유 여부를 검증해야 한다.
- 크리에이터는 키를 취소하거나 재할당할 수 있어야 한다.

### 11.3 콘텐츠 생성

사용자는 아래 형식을 업로드할 수 있어야 한다:
- 영상 (내부 저장 또는 외부 링크)
- 음성
- 텍스트
- 사진

콘텐츠에는 제목과 선택적 설명을 추가할 수 있어야 한다.  
공개 여부 설정: `공개` / `비공개`  
잠금 해제 시점 설정: 즉시 공개 또는 미래 날짜/시간 지정

### 11.4 콘텐츠 접근 제어

- 공개 콘텐츠는 크리에이터 설정에 따라 개인 키 없이 열람 가능
- 비공개 콘텐츠는 해당 키를 보유한 사용자만 열람 가능
- 시간 잠금이 걸린 비공개 항목은 유효한 키 소유자도 시점 이전에는 열람 불가
- 올바른 키가 없는 사용자는 비공개 콘텐츠를 열 수 없음
- 잠긴 항목도 크리에이터 허용 시 목록에 표시될 수 있음

### 11.5 잠긴 항목 목록 메타데이터

사용자가 콘텐츠 없이 잠긴 항목의 존재를 볼 수 있어야 하며, 표시 가능한 메타데이터는:
- 제목
- 생성일
- 잠금 해제 날짜
- 잠김/열람 가능 상태

콘텐츠 본문, 영상, 음성, 첨부파일은 권한과 시간 조건이 충족되기 전까지 접근 불가.

### 11.6 알림

- 콘텐츠가 열람 가능해지면 수신자에게 알림을 보내야 한다.
- 알림은 이메일, 푸시 알림, 또는 인앱 알림으로 전달될 수 있다.
- 크리에이터는 수신자가 자동 알림을 받을지, 수동 키 공유만 할지 선택할 수 있다.

### 11.7 크리에이터 제어

- 크리에이터는 잠금 해제 전에 콘텐츠를 수정할 수 있어야 한다.
- 크리에이터는 키를 취소하거나 재할당할 수 있어야 한다.
- 크리에이터는 콘텐츠를 삭제할 수 있어야 한다.
- 크리에이터는 콘텐츠가 공개/비공개/잠김/열람 가능 상태인지 확인할 수 있어야 한다.

### 11.8 공유 / 바이럴 요소

- 사용자는 자신의 프로필/페이지에 메시지 존재 여부를 노출할 수 있어야 한다.
- 특정인에게 남긴 메시지 목록 일부를 공개할 수 있어야 한다.
- 실제 콘텐츠는 권한이 없으면 볼 수 없어야 한다.
- 공개 범위는 비공개 / 링크 공개 / 제한 공개 등으로 제어할 수 있어야 한다.

## 12. 비기능 요구사항

- 일반 사용자도 사용할 수 있을 만큼 극도로 단순해야 한다.
- 고령자 및 비기술적 사용자를 고려한 인터페이스를 제공해야 한다.
- 보안 및 프라이버시를 최우선으로 해야 한다.
- 콘텐츠 접근 규칙은 신뢰할 수 있고 감사 가능해야 한다.
- 모바일에서 영상 및 미디어 업로드 경험이 원활해야 한다.
- 장기 콘텐츠 저장을 위한 확장성을 갖춰야 한다.
- 콘텐츠 목록 탐색 및 잠금 해제 확인이 빠르게 작동해야 한다.
- 링크 만료, 데이터 유실, 무단 접근을 방지하는 구조가 필요하다.

## 13. UX 요구사항 및 경험 원칙

### 13.1 경험 톤

프로덕트는 다음과 같은 느낌을 주어야 한다:
- 친밀하고 (intimate)
- 신뢰할 수 있으며 (trustworthy)
- 차분하고 (calm)
- 단순하며 (simple)
- 감정적으로 따뜻한 (emotionally warm)

### 13.2 UX 우선순위

- 메시지를 남기기까지 최소한의 단계
- 공개/비공개/잠긴 콘텐츠를 명확히 구분
- 각 키가 접근할 수 있는 내용에 대한 명확한 설명
- 수신자가 키를 입력하고 열람 가능한 콘텐츠를 확인하는 단순한 흐름
- 미래 메시지의 타임라인 또는 목록을 이해하기 쉬운 방식으로 제공

## 14. MVP 사용자 플로우

### 크리에이터 플로우

1. 회원가입
2. 하나 이상의 키 생성
3. 영상, 메시지, 사진 또는 음성 파일 업로드
4. 항목을 공개 또는 비공개로 표시
5. 비공개인 경우 하나 이상의 키 할당
6. 잠금 해제 날짜 설정 또는 즉시 공개
7. 저장 후 선택적으로 키 공유

### 수신자 플로우

1. 키 또는 접근 코드 수신
2. 앱/사이트 접속
3. 크리에이터 볼트 또는 수신자 페이지 접근
4. 해당 키로 허용된 항목 목록 확인
5. 아래 두 조건을 모두 충족하는 항목만 열람:
   - 해당 키에 연결된 항목
   - 시간 잠금이 해제된 항목
6. 목록 공개가 허용된 경우 잠긴 항목도 목록에서 확인 가능

## 15. 비즈니스 모델

### 15.1 무료 모델

무료 사용자는 아래와 같은 기능을 사용할 수 있다:

- 텍스트 메시지 작성
- 외부 링크 기반 영상 등록 (예: 비공개 영상 링크 등)
- 특정 날짜 예약
- 기본 수신자 지정 및 키 생성
- 링크 기반 페이지 생성
- 제한적인 저장 및 전달 기능

무료 모델의 목표: 진입 장벽 최소화, 빠른 사용자 확보, 공유/바이럴 유도, 사용성 및 감성 가치 검증

### 15.2 유료 모델

유료 사용자는 아래와 같은 프리미엄 기능을 제공받는다:

- 플랫폼 내 영상/미디어 안전 저장
- 보다 신뢰도 높은 전달 보장 프로세스
- 더 많은 메시지, 더 큰 저장 공간, 더 많은 키
- 강화된 개인정보/접근 제어
- 장기 저장 안정성 및 백업
- 다운로드/내보내기 옵션
- 고급 예약 및 수신자 관리
- 향후 선물 기능 연계 우선 제공

**유료 모델의 핵심 가치: "우리가 제대로 보관하고, 제대로 전달해준다"는 신뢰**

### 15.3 무료 vs 유료 비교

| 항목 | 무료 | 유료 |
|---|---|---|
| 텍스트 메시지 | 가능 | 가능 |
| 외부 영상 링크 등록 | 가능 | 가능 |
| 내부 안전 저장소 | 제한적/없음 | 제공 |
| 장기 보관 신뢰성 | 낮음 | 높음 |
| 고급 접근 제어 | 제한적 | 제공 |
| 백업/보안성 | 기본 | 강화 |
| 전달 보장 프로세스 | 기본 | 강화 |
| 향후 선물 기능 연계 | 제한적 | 우선 제공 |

### 15.4 향후 수익화 확장

- 기프트카드 전달 수수료
- 소형 선물 실행 수수료
- 프리미엄 스토리지 플랜
- 패밀리 볼트, 장기 보존, 고급 수신자 관리
- 고급 프라이버시 / 전달 보증 옵션
- 크리에이터/셀럽 전용 기능
- 브랜드 제휴 선물 상품
- 호스피스, 추모, 가족 지원 서비스와의 파트너십

*참고: 수익화는 감정적인 순간에 착취적이라는 인상을 주지 않아야 한다.*

## 17. 리스크 및 고려사항

### 17.1 포지셔닝 리스크
유언장/사망 서비스로 오해받을 가능성이 있다.

대응: 브랜드 메시지를 "미래 메시지", "기억", "마음 전달", "미래 예약" 중심으로 설계하고, 법률 서비스와 명확히 선 긋기

### 17.2 신뢰 리스크
서비스의 핵심은 "나중에 제대로 전달되느냐"에 대한 신뢰다.

고려사항:
- 장기 저장 방식
- 전달 실패 시 정책 (링크 만료, 플랫폼 오류 포함)
- 데이터 유실 대응
- **사업 지속성 신뢰** — 사용자는 "10년 후에도 이 플랫폼이 존재하느냐"를 우려한다

### 17.3 프라이버시 리스크
민감한 영상/텍스트가 포함될 수 있어 개인정보 및 프라이버시 보호가 중요하다.

고려사항:
- 링크 노출 위험
- 권한 관리 및 키 취소 정책
- 암호화 필요 여부
- 제3자 열람 방지

### 17.4 실행 리스크
선물 기능 확장 시 운영 복잡도가 급격히 증가할 수 있다.

고려사항: 실물 상품 재고, 가치 변동, 인플레이션, 브랜드/상품 교체, 환불 및 대체 정책  
→ 초기 단계에서는 선물 기능은 후순위 확장으로 두는 것이 적절하다.

### 17.5 바이럴과 프라이버시의 균형
존재 공개는 흥미롭지만 사용자가 부담을 느낄 수 있다.

필요 조건:
- 공개 범위 세밀한 제어
- 기본값은 보수적으로 설정
- 공개는 opt-in 중심

### 17.6 광고 및 상업화
과도한 광고 또는 상업화는 감정적 신뢰를 손상시킬 수 있다.

대응: MVP에서는 구독 기반 수익 모델에 집중하고, 광고 연동은 후순위로 둠

## 18. 가정 사항

- 사용자는 공식적인 가족 확인 시스템 없이 키 또는 코드를 공유할 의향이 있다.
- 감정적 유용성이 법적 검증보다 더 중요하다.
- 사용자는 아직 열 수 없어도 잠긴 미래 메시지의 목록을 보는 것에 가치를 느낀다.
- 공개/비공개 + 시간 기반 잠금 해제는 기존 메시징 도구와 의미 있는 차별점이다.
- 초기 제품은 커머스가 아니라 콘텐츠와 접근 제어에 집중해야 한다.

## 20. 권고사항

MVP는 하나의 명확한 약속에 집중해야 한다:

> **"지금의 마음을, 올바른 사람에게, 올바른 시점에만 열리는 메시지로 남겨라."**

프로덕트는 단순한 개인 레거시 및 시간 기반 메시지 플랫폼으로 출시되어야 하며, 법률/금융/선물 제품으로 시작해서는 안 된다. 신뢰, 사용성, 감정적 가치가 확립된 후에 수익화와 파트너십 레이어를 신중하게 추가한다.

### 전략적 권고사항 요약

1. **초기에는 유언/사망 키워드를 전면에 두지 않는다.**
   - 감정 전달, 미래 메시지, 기억 보관 플랫폼으로 시작하는 편이 대중적이다.

2. **선물 기능은 후속 확장이 적절하다.**
   - 현재 아이디어의 진짜 차별점은 메시지 전달 UX와 기대감 설계이다.

3. **유료의 본질은 기능 추가가 아니라 신뢰 판매다.**
   - 보안 저장, 전달 안정성, 책임 있는 보관을 명확히 가치화해야 한다.

4. **공개성과 프라이버시를 동시에 잡는 설계가 중요하다.**
   - "존재는 보이지만 내용은 잠김"은 강력한 감정 UX이자 바이럴 포인트다.

5. **MVP는 가볍고 명확해야 한다.**
   - 텍스트 + 외부 영상 링크 + 예약 열람 + 공개 설정 정도로 핵심을 검증하는 것이 적절하다.
